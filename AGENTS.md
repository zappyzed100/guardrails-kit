# AGENTS.md — このリポジトリ（guardrails-kit 原本）自身の開発規約

配布用テンプレートは `AGENTS.md.template`（導入先へコピーされる雛形）。本ファイルは別物で、
このリポジトリ自身を開発・編集する時にAIエージェント（Claude Code / Codex 等）が読む規約。
キット原本は自己検査で AGENTS.md/CLAUDE.md を対象外にしているが（`.guardrails-kit-source` —
`.guardrails/GUARDRAILS.md` §3.3）、原本自身の開発にも守るべき規約はあるため本ファイルを置く。

## PLAN.md の様式

ルート `PLAN.md` を新規作成・再構成する時は、`docs/plans/PLAN_FORMAT.md` が定義する節構成
（目的／アーキテクチャ／技術選定理由／評価・判断の方針／運用／ロードマップ／
タスク（機械可読））に従うこと。節の内容はリポジトリごとに書き直してよいが、骨格は崩さない。

「タスク（機械可読）」節のチェックボックス書式（`- [ ]`/`- [x]` ＋行末の状態タグ）は、
複数リポジトリの `PLAN.md` を横断収集する外部ツールが正規表現で決定的にパースする契約。
書式・書式の説明文を変更しない——変更する場合は収集側との同時変更が前提。
